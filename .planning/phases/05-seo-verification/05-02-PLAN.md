**Step 3: Implement conditional link rendering** in `components/Projects.tsx`.

Replace the `<a>` tag wrapping each project (around lines 142-146 and its closing `</a>`) with a conditional pattern:

- If `project.liveUrl` is truthy: render as `<a href={project.liveUrl}>` (same as current but using liveUrl)
- If `project.liveUrl` is falsy: render as `<div>` with the same className, data-accent, role, and onMouseEnter props, but NO href, NO target, NO rel

The approach: Create a variable for the wrapper element. Replace the `projects.map(...)` body to conditionally render `<a>` or `<div>`:

```tsx
{projects.map((project, i) => {
  const isLive = Boolean(project.liveUrl);
  const Tag = isLive ? 'a' : 'div';
  const linkProps = isLive
    ? {
        href: project.liveUrl,
        target: '_blank' as const,
        rel: 'noopener noreferrer',
      }
    : {};

  return (
    <Tag
      key={i}
      {...linkProps}
      className="project-row block group"
      data-accent={project.accent}
      role="listitem"
      aria-label={
        isLive
          ? `${project.title} — ${project.subtitle}. Visit ${project.liveUrl}`
          : `${project.title} — ${project.subtitle}`
      }
      onMouseEnter={handleRowHover}
    >
      {/* ... rest of the project card content unchanged ... */}
    </Tag>
  );
})}
```

Keep all inner content (title, subtitle, tech pills, type badge, arrow icon) exactly as-is. Only the outermost wrapper element changes from unconditional `<a>` to conditional `<a>` or `<div>`.
