## Animation Contract

This section is unique to Phase 4. It specifies all animation behaviors in prescriptive detail.

### SplitText Migration

**Hero.tsx — name character reveal:**
- Plugin: `SplitText.create()`, `type: 'chars'`, `autoSplit: true`, `onSplit` callback
- Animation: `gsap.from(self.chars, { opacity: 0, y: '140%', rotation: random ±15deg, scale: 0.5, duration: 1.0, stagger: 0.02, ease: 'back.out(2)' })`
- Replaces: `CharLine` JSX component (manual `text.split('')`)
- Scramble on role line: PRESERVE — untouched, targets `roleRef.current` via `innerText`, separate from SplitText

**About.tsx — word-by-word scroll reveal:**
- Plugin: `SplitText.create()`, `type: 'words'`, `autoSplit: true`, `onSplit` callback
- Animation: `gsap.fromTo(self.words, { opacity: 0.15, color: 'var(--text-muted)' }, { opacity: 1, color: 'var(--text)', stagger: 0.04, scrollTrigger: { scrub: 1, start: 'top 75%', end: 'bottom 40%' } })`
- Replaces: raw `innerHTML = text.split(' ').map(...)` DOM mutation
- CSS classes `.word-reveal-dim` and `.word-reveal-lit` in globals.css are retired after migration

### Architecture Diagram Path Tracing

**DrawSVGPlugin — all 3 diagrams:**
- Diagrams converted from Server Components to Client Components (`'use client'` added to `GleeMeetDiagram.tsx`, `CareerBoxDiagram.tsx`, `ZorovaDiagram.tsx`)
- Animation sequence:
  - Phase 1 (arrows draw): `tl.from(lines, { drawSVG: 0, duration: 0.4, stagger: 0.12, ease: 'power2.inOut' })`
  - Phase 2 (nodes appear): `tl.from(rects, { opacity: 0, scale: 0.85, transformOrigin: 'center center', duration: 0.35, stagger: 0.08, ease: 'back.out(2)' }, '-=0.3')`
- Trigger: `ScrollTrigger` with `start: 'top 80%'`, `once: true`
- Constraint: Each `<line>` must have explicit `strokeWidth={1}` attribute (not just CSS) for DrawSVG measurement

### Case Study Section Reveals

**CaseStudyAnimations.tsx — single client wrapper:**
- Architecture: Single client component wrapping the article body in `app/case-studies/[slug]/page.tsx`
- Page shell remains Server Component for SEO (`generateStaticParams`, `generateMetadata`, `notFound()` stay intact)
- Props: `accent: string` (CSS var name), `accentRgb: string` (RGB triple e.g. `"236,72,153"`)
- Section reveal: All elements with class `.cs-section` get `gsap.from({ opacity: 0, y: 40, duration: 0.7, ease: 'power3.out', scrollTrigger: { start: 'top 88%', once: true } })`
- Metric card reveal: `.metric-card` elements get staggered fade-up + accent flash (specification above)

### Landing Page Easing Overhaul

**Specific changes per component (from RESEARCH.md audit):**

| Component | Element | Change |
|-----------|---------|--------|
| Hero.tsx | Name chars | `elastic.out(1, 0.55)` → `back.out(2.5)` — smoother, less CPU on mobile |
| Hero.tsx | Name parallax scrub | `y: -180` → `y: -100`, `scrub: 0.6` → `scrub: 0.8` — less aggressive shift |
| About.tsx | Stats counters | `elastic.out(1, 0.5)` + `scale: 0.3` → `power3.out` + `scale: 0.85` — no jank on mobile |
| About.tsx | Skill pills stagger | `delay: i * 0.025` (15 pills = 0.375s tail) → `delay: i * 0.015`, batch cap at 8 before grouping |
| Experience.tsx | Row entrance | Remove `rotate: ±1.5deg` — imperceptible; simplify to x + opacity only |
| Contact.tsx | Form clip | Keep as-is or change clip-path direction to `inset(0 100% 0 0)` (right-to-left reveal) — Claude's discretion |

**Scrub vs. discrete trigger assignment:**
- Scrub (scroll-linked): Hero name parallax, Hero blob parallax, About word reveal
- Discrete (enter-once): All section reveals, card stagger, metric reveals, diagram animations, Experience rows, Contact elements

### Mobile Animation Strategy (gsap.matchMedia)

**Breakpoint:** `(min-width: 768px)` for desktop, `(max-width: 767px)` for mobile — matches Tailwind `md`

**Desktop (full):** Complete animations as specified above — all stagger counts, parallax, elastic easing, full durations

**Mobile (simplified):**
- No parallax effects (Hero name y-shift removed, blob parallax removed)
- Reduced stagger: max 4 children instead of full count
- Simplified easing: `power2.out` replaces all elastic/back variants
- Duration cap: 0.4s max per element (vs 0.7-1.0s on desktop)
- Word-by-word reveal still active (scrub is touch-friendly)

**Reduced motion (instant final state):**
- `gsap.set(targets, { opacity: 1, y: 0, clearProps: 'all' })` — no motion whatsoever
- Complements existing CSS rule at `globals.css:296-304` which handles CSS-driven animations
- Applies to ALL GSAP-controlled animations across all components
- Rationale: "instant final state" is the most accessible approach (prevents vestibular discomfort)

**Instance pattern:** One `gsap.matchMedia()` per component, created inside `useGSAP` callback — auto-cleaned up with component unmount.

## Interaction States

This phase introduces new animation-driven states. These are not UI states in the traditional sense but must be accounted for:

| State | Behavior | Fallback |
|-------|----------|---------|
| Pre-scroll (element not yet in viewport) | Element at `opacity: 0`, translated `y: 40px` — invisible | If JS disabled: element visible (GSAP does not run) |
| Animating (ScrollTrigger fired, mid-animation) | Element transitioning to final state | — |
| Post-animation (once: true, animation complete) | Element at full opacity/position, ScrollTrigger detached | Persistent final state |
| Diagram drawing in progress | SVG lines partially drawn (stroke-dashoffset mid-animation) | Static SVG visible if GSAP not loaded |
| Word reveal (scrub active) | Words at varying opacity 0.15–1.0 based on scroll position | All words visible if reduced-motion |
| Reduced-motion active | All elements at final state instantly; no transitions | — |
| Mobile breakpoint | Simplified animation variants as specified above | — |

## Accessibility Requirements

| Requirement | Implementation |
|-------------|---------------|
| `prefers-reduced-motion` — CSS | Already implemented in `globals.css:296-304`. No changes needed. |
| `prefers-reduced-motion` — GSAP | New: `gsap.matchMedia()` `reduceMotion` condition in every animated component. Instant final state, no motion. |
| ARIA labels on animated elements | Existing ARIA markup on all section components must be preserved through client wrapper addition — no ARIA regressions |
| Focus management | Animation must not interfere with keyboard focus. GSAP transforms do not affect focus order. `tabindex` remains unchanged. |
| Screen reader | SplitText wraps characters/words in `<span>` elements. These are inline, non-semantic spans — screen readers read through them as if they are not there. No ARIA changes needed. |
