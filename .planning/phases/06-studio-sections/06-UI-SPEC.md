## Color

Design system: "Gen Z Electric" dark palette. Source: `config/design-system.ts` + `app/globals.css`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#050505` / `var(--bg)` | Page background, section background |
| Secondary (30%) | `#161616` / `var(--line)` card bg, `#111111` elevated surface | Team cards background, section divider lines, stats area |
| Accent (10%) | `#a855f7` / `var(--violet)` | See reserved-for list below |
| Supporting accent | `#ec4899` / `var(--pink)` | Stats counter "6+" suffix, team card pill variant (pink pills) |
| Supporting accent | `#06b6d4` / `var(--cyan)` | Team card pill variant (cyan pills) |
| Text primary | `#fafafa` / `var(--text)` | Headings, card names, stat numbers |
| Text secondary | `#a1a1aa` / `var(--text-secondary)` | Narrative body, card role, methodology body |
| Text muted | `#52525b` / `var(--text-muted)` | Stat labels, section labels |
| Destructive | `hsl(0 84% 60%)` / `var(--destructive)` | Not applicable in this phase (no destructive actions) |

**Accent (`--violet`) reserved for:**
1. Section label text color (e.g., `01 / About`)
2. Counter suffix (`+`) on "8+ Combined Years" stat
3. Avatar circle background on team cards (both founders — same color per D-07)
4. Methodology step numbers
5. Border-left on active/hover states if introduced in team card
6. Focus ring (`outline: 2px solid var(--violet)`) — already global

**Rule:** Accent must NOT be used for body text, general hover states on non-interactive elements, or decorative dividers. Pink and cyan are supporting accents reserved for pill color variants only.

## Interaction Contracts

### Stats Counters (About.tsx)
- Pattern: identical to existing `About.tsx` counter approach (IntersectionObserver + setInterval)
- Three counters: `8` (suffix `+`, violet), `6` (suffix `+`, pink), `50` (suffix `+`, cyan)
- Counter labels: `"Combined Years"`, `"Products Shipped"`, `"Technologies"`
- Fire once on scroll entry, `threshold: 0.1`
- Reduced-motion: show final value instantly, skip counting animation

### Team Card Social Links
- `<a>` with `target="_blank" rel="noopener noreferrer"`
- `aria-label`: `"Abhishek Vaghela on GitHub"`, `"Abhishek Vaghela on LinkedIn"` (etc. per member)
- Hover: `text-[var(--text-muted)] hover:text-white transition-colors duration-300`
- No card-level hover lift — per D-06 (no hover glow)
- Focus-visible: inherits global `outline: 2px solid var(--violet)`

### Methodology Steps
- No interactive state — static typographic blocks
- No accordion, no expand/collapse
- Scroll-triggered reveal only (see Animation Contracts below)

## Animation Contracts

All animations use `useGSAP` + `gsap.matchMedia()`. Pattern established in Phases 1–4. Every animated component must declare `reduceMotion` branch that calls `gsap.set(..., { clearProps: 'all' })` for instant final state.

### About.tsx Animations

| Element | Animation | Duration | Ease | Trigger |
|---------|-----------|----------|------|---------|
| Section label | clip-path reveal `inset(0 100% 0 0)` → `inset(0 0% 0 0)` | 1s | `power3.inOut` | `top 90%` |
| Section heading | `y: 60 → 0, opacity: 0 → 1` (desktop); `y: 30 → 0` (mobile) | 1s / 0.4s | `power3.out` | `top 85%` |
| Narrative paragraph | SplitText word-by-word, `opacity: 0.15 → 1, color: --text-muted → --text` | scrub 1 | scroll-scrub | `top 75%` / `bottom 40%` |
| Stat numbers | `scale: 0.85 → 1, opacity: 0 → 1, y: 20 → 0`, stagger `i * 0.1` | 0.6s / 0.3s | `power3.out` | `top 90%` |
| Team cards | `x: 40 → 0, opacity: 0 → 1`, stagger `0.12s` between cards | 0.8s / 0.4s | `power3.out` | `top 85%` |
| Skill pills | `scale: 0 → 1, opacity: 0 → 1`, stagger `0.015s` (capped at 8 items) | 0.35s / 0.2s | `back.out(3)` / `power2.out` | `top 95%` |

Selector pattern for team cards: `.team-card` class on each card element.

### Methodology.tsx Animations

| Element | Animation | Duration | Ease | Trigger |
|---------|-----------|----------|------|---------|
| Section label | clip-path reveal (same as all sections) | 1s | `power3.inOut` | `top 90%` |
| Section heading | `y: 60 → 0, opacity: 0 → 1` | 1s / 0.4s | `power3.out` | `top 85%` |
| Step blocks | `y: 40 → 0, opacity: 0 → 1`, stagger `0.15s` between steps | 0.8s / 0.4s | `power3.out` | `top 80%` |

Reduced-motion branch for both components: `gsap.set` all animated selectors to `opacity: 1, x: 0, y: 0, scale: 1`, then `clearProps: 'all'`. No transitions, no delays.
