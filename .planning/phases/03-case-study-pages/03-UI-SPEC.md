## Spacing Scale

Declared values (multiples of 4 — 8-point grid):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, pill padding, inline gaps |
| sm | 8px | Tight element spacing, gap between pills |
| md | 16px | Default element spacing, section content gaps |
| lg | 24px | Card internal padding, subsection spacing |
| xl | 32px | Section-to-section separation within a page |
| 2xl | 48px | Major section breaks, architecture diagram margins |
| 3xl | 64px | Page-level vertical rhythm between case study sections |
| hero-pt | 128px (pt-32) | Top padding under fixed Navigation — all page shells |

Exceptions:
- Touch targets for CTA buttons: minimum 44px height
- Pattern Spotlight callout block: 24px horizontal + 16px vertical inner padding (asymmetric for visual weight — both values on 4px grid)
- Architecture diagram container: no max-width constraint — full bleed within the max-[1400px] shell
- Section numbering label (`01 /`): 0 left margin, dev-mono 10px, tracks left edge

## Typography

All sizes use clamp() where noted — these are the declared minimum/target/maximum. The two system weights are **400 (regular)** and **700 (bold)**. Utility-specific weight overrides are noted per row and do not constitute additional system weights.

| Role | Size | Weight | Line Height | Font | Class / Token |
|------|------|--------|-------------|------|---------------|
| Body | clamp(1rem → 1.15rem) | 400 (regular) | 1.75 | Inter | `.text-body` (existing global utility) |
| Label / mono | 10–12px (0.625–0.75rem) | 400 (regular); 500 permitted within this utility only | 1.4 | JetBrains Mono | `.dev-mono` (existing global utility) |
| Section / sub-heading | clamp(1.25rem → 2rem) | 700 (bold) | 1.1 | Inter | `.heading-md` (existing global utility) |
| Page heading | clamp(2rem → 4rem) | 700 (bold) | 1.0 | Inter | `.heading-lg` (existing global utility) |

Notes:
- `.heading-brutal` (clamp(3.5rem → 12rem), weight 900, line-height 0.88) is a display utility used optionally on the index pattern intro section. Weight 900 is confined to this utility and is not a type system weight. It is not assigned a row in the scale table.
- The two declared system weights are 400 and 700. Weight 500 is a permitted variance within `.dev-mono` only. Weight 900 is confined to `.heading-brutal` only. No other weight values are in use in this phase.

## Page-Level Layout Specifications

### Index Page (`/case-studies`)

**Shell:** Server Component, max-w-[1400px] mx-auto, px-6 lg:px-16 xl:px-24, pt-32 pb-20 (matches existing skeleton)

**Section order:**
1. **Page header** — mono label "Case Studies", h1 "How we build.", body subtitle (2-line max)
2. **Pattern intro block** — full-width card, `var(--bg-elevated)` background, cyan accent border-left 3px. Copy: "Our Signature Stack" heading + 2-sentence explanation of Redis + OpenSearch + DB pattern. Three inline pills: `pill-cyan` for "Redis", "OpenSearch", and the project-specific DB.
3. **Case study cards grid** — 3 cards in a vertical stack (not a 3-column grid — long editorial card per D-07). Each card: title, tagline, tech pills (top 4), project type badge, accent color reflected in top border (4px) or left border stripe. Click = navigate to `/case-studies/[slug]`.
4. **Bottom rule** — `border-t border-[var(--line)]`

**Index card anatomy (each card, top to bottom):**
- Project type badge (top right): `pill-{accent}` with "client" / "co-built" text
- Project number: dev-mono, muted, left-aligned (`01`, `02`, `03`)
- Title: `.heading-md` weight 700, white
- Tagline: `.text-body`, 1 line max on desktop (truncate with ellipsis on overflow)
- Tech pills: 4 pills max, `pill-{project.accent}` color
- "Read Case Study" link: small, dev-mono, accent color, ArrowRight icon

### Detail Page (`/case-studies/[slug]`)

**Shell:** Server Component (page shell), Client Components for GSAP-animated sections. max-w-[1400px] mx-auto, px-6 lg:px-16 xl:px-24, pt-32 pb-20.

**Section anatomy — all 8 sections follow this pattern:**
- Section number label: dev-mono 10px, `var(--{project.accent})`, position top-left, format `01 /`
- Section heading: `.heading-md`, white, weight 700
- `border-t border-[var(--line)] pt-12 mb-20` between each section
- Body: `.text-body` for narrative prose

**Section order (per D-02):**

| # | Section | Visual treatment |
|---|---------|-----------------|
| 01 | Hero | Full-width. Title `.heading-lg`, tagline `.text-body max-w-3xl`. Tech pills row. Accent backdrop: radial gradient from `var(--{accent})` at 5% opacity, top-right quadrant. |
| 02 | Challenge | Narrative prose. 2-3 paragraphs. No special visual treatment — text-first. Max-w-prose (65ch). |
| 03 | Architecture Diagram | Full-width SVG component (React SVG inline). Dark background card (`var(--bg-elevated)`), 24px padding, border `var(--line)`, border-radius `var(--radius)`. Contains Pattern Spotlight callout block below diagram. |
| 04 | Tech Rationale | Three-column grid on desktop (`sm:grid-cols-1 lg:grid-cols-3`). Each card: technology name (mono), problem statement, solution — structured from `techRationale[]` typed array. Card bg `var(--bg-elevated)`, border `var(--line)`. |
| 05 | Implementation Highlights | Numbered list of 3-5 highlights. Each item: number (dev-mono accent color), bold title, body detail. Not a bullet list — visual weight with the number as an accent element. |
| 06 | Results / Metrics | 2-4 metric callout cards in a 2-column grid (lg:grid-cols-4 for 4 items). Each card: metric label (dev-mono muted uppercase), impact statement (`.heading-md` white), description (`.text-body` small). Per D-03: qualitative only. |
| 07 | Team Attribution | Horizontal row of team member pills + role. `pill-cyan` for names. Short sentence per person on their contribution. |
| 08 | CTA | Centered. Heading "Want similar results? Let's talk." Body: project-specific one-liner. `.magnetic-btn` white button "Get in Touch" → `/#contact`. Violet glow: `box-shadow: 0 0 40px rgba(168,85,247,0.15)` on the button container. |

## Pattern Spotlight Callout Block

This is a cross-cutting component appearing in the Architecture section of every detail page. It must be visually identical across all 3 case studies (consistent studio signature).

**Anatomy:**
- Left border: 3px solid `var(--{project.accent})`
- Background: `rgba(255,255,255,0.02)` over `var(--bg-elevated)`
- Inner padding: 16px vertical, 24px horizontal
- Label: dev-mono 10px uppercase "Pattern Spotlight" in `var(--{project.accent})`
- Heading: 16px weight 700 white — "Redis + OpenSearch + {DB name}" (project-specific DB)
- Body: `.text-body` — 2-3 sentences explaining how the pattern is applied in this specific project's context

---

## Architecture Diagram Specification

Per D-06: 5-8 boxes max. High-level system overview. Boxes represent major layers, arrows show data flow.

**Implementation:** Inline React SVG component (`.tsx` file, e.g., `components/diagrams/GleeMeetDiagram.tsx`). Static SVG — no animation in Phase 3 (animations deferred to Phase 4 per ANIM-03).

**Visual rules:**
- Canvas: 800×400px viewBox, scales via `width="100%" height="auto"` — responsive
- Background: transparent (parent card provides bg)
- Node boxes: 120×48px rounded-rect (`rx="8"`), fill `var(--bg)`, stroke `var(--line)`, stroke-width 1.5
- Node text: JetBrains Mono, 11px, fill `var(--text-secondary)`, centered
- Accent nodes (Redis, OpenSearch, primary DB): stroke `var(--{project.accent})`, fill `rgba({accent-rgb},0.08)`
- Arrows: `var(--line-light)` color, stroke-width 1, marker-end arrowhead
- Layer labels (Client / API / Cache / Search / Database): dev-mono 9px uppercase, fill `var(--text-muted)`, positioned above each column

## Interaction Contracts

### Index Page Interactions

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Card hover | Mouse enter case study card | Title color transitions to `var(--{project.accent})`, border-top transitions from `var(--line)` to `var(--{project.accent})`, duration 300ms ease |
| Card click | Click anywhere on card | Navigate to `/case-studies/[slug]` — standard Next.js Link |
| Card focus | Keyboard tab | Same visual state as hover; `outline: 2px solid var(--violet)` focus ring |
| "Read Case Study" arrow | Mouse enter | Arrow translates `translate(4px, -4px)` in 200ms |

### Detail Page Interactions

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| CTA button hover | Mouse enter `.magnetic-btn` | `transform: scale(1.04)` — existing `.magnetic-btn` class handles this |
| CTA button active | Mouse down | `transform: scale(0.97)` — existing `.magnetic-btn` class handles this |
| CTA button click | Click "Get in Touch" | Navigate to `/#contact` (cross-route hash nav via `handleNavClick` pattern from Phase 2) |
| Tech pill hover | Mouse enter any pill | No state change — pills are display-only, not interactive |
| External project link (if present in hero) | Click | `target="_blank" rel="noopener noreferrer"` — open project URL in new tab |

### Reduced Motion

All GSAP animations deferred to Phase 4. Phase 3 ships CSS transitions only. All `transition-*` durations reduce to 0.01ms under `@media (prefers-reduced-motion: reduce)` — handled by existing global rule in `app/globals.css`.
